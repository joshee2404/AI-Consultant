Layer 3 — Orchestration (Sequential Pipeline · 4 Stages)
Stage 1 — Pattern Detection
🔍
Analysis Engine
services/analysis_engine.py
Loads prompts/analysis_prompt.py. Sends structured prompt to Claude. Receives JSON: pattern types per process (repetitive / manual-decision / data-heavy), AI use case category, impact estimate, rationale per opportunity.
Claude 3.5 Sonnet
→ Output: Opportunity List (raw JSON)
opportunities[]
🧲
Embedding Validator
services/embedding_service.py
Embeds each process description using Titan. Cosine-similarity match against 5 pre-embedded AI category vectors. Validates / corrects Claude's use case mapping. Ensures semantic accuracy.
Titan Embeddings v1
→ Output: Validated Opportunities + Similarity Scores
Stage 2 — Complexity Scoring
🧮
Complexity Scorer
services/complexity_scorer.py
Pure rule-based Python. No AI. Scores each opportunity on 3 axes from form data: data availability + integration dependency + real-time requirement. Sums to Low / Medium / High classification.
→ Output: Complexity Score per Opportunity
scored list
📐
Matrix Builder
utils/matrix_builder.py
Takes complexity (from scorer) + impact (from Claude) per opportunity. Builds 2D grid data structure for the Impact vs Complexity scatter matrix chart.
→ Output: Impact/Complexity Matrix Data
Stage 3 — Report Generation
📝
Report Generator
services/report_generator.py
Loads prompts/report_prompt.py. Sends scored + validated opportunities to Claude. Claude generates: executive summary prose, 3-phase roadmap, tech stack per opportunity. Second and final Claude call.
Claude 3.5 Sonnet
→ Output: Executive Summary + Phased Roadmap + Tech Stack
full report
🔗
Report Assembler
utils/report_assembler.py
Merges all pipeline outputs into one final report JSON object validated against models/output_schema.py. Assigns report_id and timestamp.
→ Output: Final Assembled Report JSON
API calls via bedrock_client.py → AWS Bedrock
Layer 4 — AI Models (AWS Bedrock)
🧠
Claude 3.5 Sonnet
anthropic.claude-3-5-sonnet-20240620
Called twice. Call 1 (analysis): pattern identification + use case mapping + impact scoring. Call 2 (report): executive summary + phased roadmap + tech recommendations.
→ Call 1: Opportunity List JSON
→ Call 2: Report Narrative JSON
🔢
Amazon Titan Embeddings
amazon.titan-embed-text-v1
Called once per assessment. Embeds process descriptions. Pre-embedded category vectors loaded at startup. Cosine similarity validates AI use case classification from Claude.
→ Output: 1536-dim Similarity Vectors
📄
Prompt Templates
prompts/analysis_prompt.py
prompts/report_prompt.py
System prompt + user prompt templates. Injected with form data before each Claude call. Enforce JSON-only response format with explicit schema.
→ Input: Structured Prompts to Claude
Assembled Report JSON → Persist to Database
Layer 5 — Database (PostgreSQL / DynamoDB)
💾
Assessments Table
db/models/assessment.py
Stores: report_id, company_name, industry, created_at, status (processing / complete / failed), raw form input JSON.
→ Stores: Input Record
foreign key
📦
Reports Table
db/models/report.py
Stores: full assembled report JSON, readiness score, opportunity count, phase count. Linked to assessment by report_id. Retrieved on GET /report/:id.
→ Stores: Full Report JSON
links to
🔧
DB Manager
db/database.py
SQLAlchemy session factory (or DynamoDB client). Exposes save_report(), get_report(id), update_status(). Used by orchestration layer.
→ CRUD: Read / Write Operations
Final Report JSON → Output Generation
Layer 6 — Output (Deliverables)
📋
AI Opportunity List
components/OpportunityCard.jsx
Generated by: Analysis Engine + Embedding Validator. Each card shows: process name, AI use case category, impact level, rationale, and suggested tech stack.
→ Source: Claude Call 1 + Titan
🗺️
Impact vs Complexity Matrix
components/ImpactMatrix.jsx
Generated by: Matrix Builder. Recharts scatter plot. X = Complexity (Low→High), Y = Impact (Low→High). Each dot = one AI opportunity. Color-coded by use case type.
→ Source: matrix_builder.py
🗓️
Phased Roadmap
components/PhasedRoadmap.jsx
Generated by: Report Generator. 3-phase horizontal timeline. Phase 1: quick wins. Phase 2: core build. Phase 3: intelligence layer. Each phase lists its opportunities + milestone.
→ Source: Claude Call 2
🏗️
Tech Stack Recommendations
components/TechStack.jsx
Generated by: Report Generator. Per-opportunity and overall recommended AWS + AI tech stack. Considers existing tools from form input to avoid redundant stack suggestions.
→ Source: Claude Call 2
📜
Executive Summary
components/ExecutiveSummary.jsx
Generated by: Report Generator. Auto-generated prose paragraph. Covers: readiness score, top 3 opportunities, total time savings estimate, recommended first action, GDPR flags if any.
→ Source: Claude Call 2
⬇️
PDF Export
utils/pdf_export.py
Server-side PDF generation via WeasyPrint / ReportLab. Formats all 5 outputs into a branded downloadable report. Triggered by GET /report/:id/pdf.
→ Source: Full Report JSON → .pdf
